# Render: staging vs production (Fikiri backend)

- **Staging** = service `fikiri-backend-staging`, Git branch **`develop`**, `FLASK_ENV=staging`, default `DATABASE_URL` in blueprint is SQLite at `sqlite:///data/fikiri_staging.db`, lower `WEB_CONCURRENCY` / `GUNICORN_THREADS` (2 vs 4).
- **Production** = service `fikiri-backend`, branch **`main`**, `FLASK_ENV=production`, `DATABASE_URL` set in dashboard (Postgres), disk mount for `/opt/render/project/data`, stricter rate limits, `GOOGLE_REDIRECT_URI` for production domain.

Same build/start pattern for both web apps: `pip install -r requirements.txt` + gevent-websocket Gunicorn worker on `wsgi:wsgi_app`, `SOCKETIO_ASYNC_MODE=gevent`, health `/api/health/live`.
