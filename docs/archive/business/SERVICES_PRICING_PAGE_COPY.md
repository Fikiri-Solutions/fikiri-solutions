Software subscriptions cover the platform. **Implementation services** cover the work that turns your inbox and CRM into a reliable system: mapping, cleanup, automation build, training, and a defined stabilization window. We scope work so you get **concrete deliverables** at each step—not open-ended consulting.

## Principle strip (3 bullets — “why us”)

- **Tangible deliverables at every phase** — You receive documents, maps, scorecards, and a live workflow—not slides about “digital transformation.”  
- **Foundations before automation** — We do not automate broken CRM rules or mystery data; that is how rework gets baked in.  
- **Build once** — Correct routing, ownership, and handoffs the first time, so you are not paying twice for the same problem.

## Package cards — copy blocks

### Package 1 — Workflow Diagnostic

**One sentence:** We pick **one** workflow, map it, score it, and give you a conservative savings estimate plus a clear “do this next / not yet” list.

**You walk away with:**

- Process map (current → future)  
- Automation scorecard (volume, structure, risk)  
- Savings estimate (hours + dollars, assumptions on one page)  
- Recommended module stack for build  
- “Not yet” list (what we will not do next—and why)

**Client time:** ~2 hours.  
**CTA button:** `Book diagnostic` / `Get a quote`

**CTA button:** `Start foundations` / `Talk to us`

### Package 3 — Automation Build Sprint

**One sentence:** We ship **one** workflow end-to-end—tested on real examples, with training, handoff checklist, rollback plan, and **30-day stabilization** for in-scope issues.

**You walk away with:**

- Production automation for the scoped workflow  
- Test walkthrough on agreed examples  
- Training for named roles  
- Handoff checklist + rollback / pause plan  
- 30-day stabilization (in-scope fixes)

**Typical window:** 2–4 weeks build + stabilization.  
**CTA button:** `Scope my build` / `Book a call`

## FAQ snippets (optional)

**Why not skip straight to automation?**  
Automating bad data or unclear ownership ships fast—and breaks often. Foundations exist so we build **once**.

**What counts as “done”?**  
Each package lists **deliverables you can save and use**. Package 3 adds UAT sign-off and a 30-day stabilization window for in-scope issues.

**What about AI?**  
AI layers come **after** inputs and routing are stable—priced as a separate phase so cost and quality stay predictable.

**What happens after 30 days?**  
Ongoing tuning and new requests use a **monthly care plan** or a new scoped project—so support is never unlimited and unpaid by accident.
